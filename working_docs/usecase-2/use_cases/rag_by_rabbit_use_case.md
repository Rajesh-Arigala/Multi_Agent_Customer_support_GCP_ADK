# RAG by RABBIT Use Case

## Purpose
This document defines use case 2: RAG by RABBIT, expanded as RABBIT Agent GALA. It is the action-oriented evolution of RABBIT Assistant for `rajesharigala.com`.

## Audience
- Rajesh Arigala
- Recruiters, HR teams, hiring managers, consultants, collaborators, and business stakeholders
- Developers implementing the action-agent layer
- Future operators reviewing engagement leads and opportunity tickets

## Name
Public-facing name:

```text
RAG by RABBIT
```

Expanded/internal name:

```text
RABBIT Agent GALA
```

Meaning:

```text
RAG = Retrieval-Augmented Generation
RAG = RABBIT Agent GALA
GALA = wordplay from AriGALA
```

## Intent
RAG by RABBIT should turn Rajesh Arigala's professional website into a living proof-of-work opportunity engine.

It should:

- answer with evidence from `rajesharigala.com` and the RABBIT RAG corpus
- show that the agent itself was built by Rajesh Arigala
- qualify stakeholder intent
- route national and international money-capital and social-capital opportunities
- capture serious conversations as engagement tickets

## Core Flow
```text
Visitor lands on rajesharigala.com
-> asks about Rajesh's business-tech profile
-> RAG by RABBIT answers from website/project evidence

-> agent collects lightweight contact/context details
```

## Examples
### Hiring
```text
User: We are hiring for an AI transformation role. Is Rajesh suitable?
Agent: Answers from evidence, explains role fit, and asks whether the visitor wants to share opportunity details.
Outcome: Hiring / Job Opportunity ticket.
```

### Consulting
```text
User: We need help building a RAG system for our company.
Agent: Collects company, problem, timeline, preferred discussion mode, and contact details.
Outcome: Consulting Opportunity ticket.
```

### Speaking Engagement
```text
User: Can Rajesh speak at our AI meetup?
Agent: Collects event name, topic, audience size, date, location/online, honorarium if available, and organizer contact.
Outcome: Speaking Engagement ticket.
```

### Third-Party Presentation
```text
User: Can Rajesh present RABBIT or Sentinel to our leadership team?
Agent: Collects organization, audience, topic, time window, format, and decision-maker contact.
Outcome: Presentation To Third Parties ticket.
```

## Living Proof Rule
The visitor should understand that the agent itself is part of Rajesh's proof of capability.

Allowed framing:

```text
You are speaking with RAG by RABBIT, an action-oriented professional opportunity agent built by Rajesh Arigala. The system itself is live proof of his ability to design RAG, workflow-agent, action-agent, and business-facing GenAI systems.
```

## Decisions
- Keep RABBIT Assistant as the knowledge/RAG layer.
- Use RAG by RABBIT as the action-agent layer.
- Position opportunities as national and international.
- Track both money capital and social capital.
- Capture serious stakeholder conversations as engagement tickets.
- Connect this use case later to Sentinel Gateway as the enterprise control-plane layer.
